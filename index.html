Treq

Test | Request | Encrypt | Query

Welcome to your own HTTPS training lab in a container.

What is this?

This project is a hands-on sandbox to explore the difference between HTTP and HTTPS, understand TLS encryption, and simulate a basic Man-in-the-Middle logging scenario.

Objectives

  • Compare file downloads over HTTP and HTTPS
  • Observe what a "public" zone feels like vs a secure one
  • See how credential leakage happens on insecure channels
  • Use tools like curl -v and openssl s_client to inspect handshakes

Explore These Paths

  • /files/ – Mixed test files
  • /public/ – Unsecured public zone
  • /admin/ – Forbidden over HTTP
  • /robots.txt – Sneaky clues

Find All the Flags

As you explore, you'll find hidden FLAG{} values. Collect them.
Once you have them all, go to /submit.html to unlock the easter egg.

Terminal Integration

The secret command you’ll get will have use on my portfolio website.

Tools to Try

  • curl -v https://treq.test/public/help.txt
  • curl -v http://treq.test/files/plainview.txt
  • curl -v https://treq.test/files/plainview.txt
  • openssl s_client -connect treq.test:443

Pro Tips

  • Use a browser to inspect certificate warnings.
  • Try accessing HTTP-only zones over HTTPS and vice versa.
  • Compare what changes when using a secure vs insecure protocol.
