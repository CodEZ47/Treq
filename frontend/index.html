Treq

Test | Request | Encrypt | Query

A hands-on HTTPS, TLS, and reverse proxy security lab in a container.

What is Treq?

Treq is a self-contained, multi-container learning environment that teaches modern web security concepts through real challenges. You'll inspect certificates, intercept network traffic, and decrypt secrets using the same tools real security engineers use every day.

Get Started

The fastest way to begin: drop into the attacker container that comes preloaded with mitmproxy and other tools. 

docker exec -it treq-hacker bash

Then submit the flags you find at /submit.html.

The Challenges

Flag 1 — Certificate Inspection Available

Every HTTPS connection starts with a certificate. Inspect the one this server is using and you'll find a flag hidden inside.

Hint: openssl s_client is a great starting point.

Flag 2 — HTTP Credential Interception Available

Background traffic is being sent across the network right now. Some of it shouldn't be in plaintext. Intercept it, find the leaked credentials, and use them on the admin panel.

Hint: Run mitmproxy --listen-port 8080 --listen-host 0.0.0.0 inside the hacker container.

Flag 3 — Information Disclosure Coming Soon

Public files reveal more than they should. Find the breadcrumb trail.

Flag 4 — Access Control Bypass Coming Soon

HTTP and HTTPS treat the same resource differently. Find out why that matters.

Flag 5 — Mixed Content Coming Soon

One insecure asset can compromise an otherwise secure page.

Useful Commands

  • openssl s_client -connect treq.test:443 — inspect TLS handshake and certificate
  • curl -v https://treq.test — verbose HTTPS request, see headers and TLS info
  • docker exec -it treq-hacker bash — drop into the attacker container
  • docker logs treq-frontend — see what the server is logging

Treq is an educational lab. The goal is learning, not winning. Solving challenges properly teaches concepts that apply to real production security.
